ℹ️ About MedAI

Project overview, tech stack & API reference

🧬

Multi-Disease Prediction System

End-to-end ML application predicting Diabetes & Heart Disease risk using trained models on your real clinical datasets. The standalone UI works with client-side fallback logic, while the Flask backend serves the latest saved artifacts and now prefers the optimized heart pipeline for live heart predictions.

🔬 Datasets

Diabetes Prediction Dataset — 100,000 samples with demographic, lifestyle, and lab features such as age, BMI, HbA1c, and blood glucose.

Cardio Train Dataset — 68,742 cleaned samples with blood pressure, anthropometric, cholesterol, glucose, and activity indicators.

Preprocessing: categorical encoding, BMI derivation, outlier filtering, StandardScaler normalisation, and stratified train-test split.

🤖 ML Pipeline

1 — Load CSV & clean data
2 — Encode categories and derive BMI where needed
3 — Filter invalid blood pressure and anthropometric outliers
4 — StandardScaler normalisation
5 — 80/20 stratified split
6 — Train LR + RF (+ XGBoost if installed)
7 — Run the optimized heart pipeline with feature engineering, class balancing checks, and tuning
8 — Save the best model, scaler, metrics, and optimized heart artifacts

🚀 Setup
# 1. Install
pip install -r requirements.txt

# 2. Train baseline models
cd backend && python train_models.py

# 2b. Optional: rebuild the optimized heart pipeline used by the API
python train_heart_optimized.py

# 3. Start API
python app.py   # → http://localhost:5000

# 4. Open frontend
open ../frontend/index.html
# Works offline too — uses embedded scaler params and client-side fallback predictions
